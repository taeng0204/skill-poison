[Slide 12] Asymmetric Prevention Burden — User as Sole Gatekeeper
────────────────────────────────────────────────────────────

[Slide 15] Research Questions — Four Core RQs
────────────────────────────────────────────────────────────
"위 공백을 4개 RQ로 정리했습니다.

각 RQ는 측정 가능한 정량 지표와 1:1 매핑되어 있습니다."

[Slide 17] The 4-Stage Closed-Loop Pipeline
────────────────────────────────────────────────────────────

[Slide 23] Models (2/2) · XAI Methods Considered
────────────────────────────────────────────────────────────
"두 번째 — XAI 방법론입니다. 4개 후보를 모두 평가할 계획입니다.
LIME은 local perturbation으로 per-skill phrase 단위 attribution,
SHAP은 Shapley-value 기반 정량적 per-feature attribution,
Counterfactual은 결정을 뒤집는 최소 입력 변경 — 그게 곧 방어 룰이
되고요,
Attention visualization은 attribution은 아니라 보조 지표로 활용합니다.

최종 선택은 attribution stability와 Stage 04 방어 설계 활용도를
기준으로 결정합니다. 현재 prototype은 Decision Tree + Permutation
Importance를 baseline으로 사용 중입니다."
(60초)

────────────────────────────────────────────────────────────
[Slide 25] Stage 04 — Comparing Two Defenses + Closed-Loop Validation
────────────────────────────────────────────────────────────
"Stage 04는 두 방어 기법을 비교합니다.
A · Self Guideline — Skill 로딩 전 모델에게 안전 체크 규칙을 주입해서
모델이 스스로 위험 패턴을 거부하도록 하는 방식.
B · Proxy Hooks — Skill 텍스트를 모델에 전달하기 전 외부 hook이
정규식·AST·credential density 기반으로 차단·경고하는 방식.
검증은 Stage 02 시뮬레이션을 A·B 각각 적용한 환경에서 재실행해서
ASR 감소량으로 직접 비교합니다."
(55초)
