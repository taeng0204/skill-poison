Threat · 메커니즘

Prompt Injection vs. Trusted Skill Poisoning

공격 표면이 외부 데이터에서 시스템 프롬프트로 이동

기존: Prompt Injection

외부 데이터 → 비신뢰 영역

기업의 LLM 서비스를 외부 사용자가 우회 시도. Instruction Hierarchy(시스템 > 유저 > 외부)로 방어 가능.

· 공격 주체: 외부 사용자
· 공격 표면: 데이터 입력단
· 방어 수단: 계층적 신뢰 모델

본 연구: Trusted Skill Poisoning

Skill 파일 → 시스템 프롬프트 주입

개발자가 유용해 보여 다운로드한 Skill이 신뢰된 시스템 영역으로 그대로 주입됨.

· 공격 주체: 마켓·OSS 공급자
· 공격 표면: Skill 파일 자체
· 방어 수단: 구조적 공백

차별점
사용자 자신이 게이트키퍼가 되는 구조이며, 자동 주입 메커니즘이 검증을 우회한다.

Threat · 책임 구조

예방 책임의 비대칭 — 사용자 단일 게이트키퍼

공급자·플랫폼·모델 어디에도 검증 의무가 없고, 책임은 사용자 한 명에게 수렴한다

  • Skill 공급자는 검증 의무 없음 (오픈 마켓·OSS 모델)
  • 플랫폼은 자동 주입까지만 책임
  • 모델 자체의 안전성은 일관되지 않음 — 모델별 큰 편차
  • 최종 검증·결정 책임이 개별 사용자에게 집중
현재 구조의 한계
사용자 = 단일 게이트키퍼
수만 줄의 Skill 텍스트를 매번 검토할 수 있는 사용자는 없다.
자동 검증 도구의 부재가 위협 확산의 구조적 원인.
연구 동기
Skill 특성과 위험성의 정량 매핑이 가능하다면, 가이드라인·proxy 도구로 자동 검증 가능.

Research Gap

선행연구가 멈춘 지점 — 본 연구가 채울 3가지 공백

정적 탐지 · 생태계 분석은 성숙. 그 이후의 실험적 측정은 미수행

GAP 01 · Agent 비교
Agent별 Skill 수용 차이 측정 부재
Snyk·SkillSieve·DDIPE는 skill 텍스트 자체를 분석하거나 단일 환경 bypass rate만 측정. Closed agent (Claude Code · Codex) vs Open agent (OpenCode + open LLM) 직접 비교 미수행.
GAP 02 · 단계 분리
결정 단계별 ASR 분리 측정 부재
기존 연구는 단일 ASR 또는 detection F1 보고. Acceptance(수용) → Execution(실행) → Egress(유출) 세 단계로 분리한 ASR 실험 미수행.
GAP 03 · 방어 비교
방어 기법 closed-loop 효과 비교 부재
탐지 도구는 정확도만 평가. 같은 공격 셋을 같은 환경에서 두 방어 기법으로 재실행한 비교는 없음. Self Guideline vs Proxy Hooks 직접 비교가 필요.
한 줄 요약
Skill 탐지·생태계 분석은 활발하나, 단계별 ASR 측정 + Agent 비교 + 방어 closed-loop 비교는 아직 다뤄지지 않음.

Research Questions

연구 질문 — 4가지 핵심 RQ

각 RQ는 측정 가능한 정량 지표와 1:1 매핑

RQ1
각 AI Agent는 악성 Skill을 얼마나 수용·차단하는가?
측정: agent별 block / review / restricted-allow / allow 비율
RQ2
Agent가 차단할 때 어떤 근거를 제시하는가? 그리고 그 설명은 실제 행동과 일치하는가?
측정: decision_reason · matched_defense_signal · reason-action consistency
RQ3
동일 Skill에 대해 Agent별 판단 차이는 어떻게 나타나는가?
측정: agent 간 decision disagreement · false-allow 사례 분석
RQ4
어떤 방어 기법이 악성 Skill을 막는 데 더 효과적인가? — Self Guideline vs Proxy Hooks
측정: 두 방어 기법 ASR 감소 비교 · acceptance / execution / egress 단계별 차단 비율

Credential 변수는 결정에 영향을 주지 않음

XAI 모델: Decision Tree (max_depth=4) + Permutation Importance (n_repeats=30, balanced_accuracy)

시사점

객관적 credential density에 기반한 외부 risk scorer가 보완 장치로 필요.
