[Slide 06] Prompt Injection 연구 흐름과 본 연구의 위치
────────────────────────────────────────────────────────────
"학계 흐름을 짧게 정리하면 —
2023년 Greshake et al.이 Indirect Prompt Injection을 처음 실증했고,
2024-25년 AgentDojo, InjecAgent 같은 agent 환경 PI 평가 연구가 나왔고,
2025-26년 CaMeL, RTBAS, Progent, ASIDE 같은 방어 메커니즘이 제안됐습니다.

저희 연구는 가장 오른쪽 — Trusted Skill Poisoning. 시스템 프롬프트로
직접 주입되는 Skill 자체의 Privacy 위협으로, 아직 체계적으로 다뤄지지

[Slide 07] Prompt Injection vs Trusted Skill Poisoning
────────────────────────────────────────────────────────────
"두 공격을 비교하면 — 왼쪽 기존 PI는 외부 사용자가 기업 서비스를

오른쪽이 저희 연구 — 사용자가 직접 다운로드한 Skill이 이미 신뢰된
시스템 레벨로 주입돼버립니다. 사용자 자신이 게이트키퍼인데, 자동
주입 메커니즘이 그 검증을 우회한다는 게 차별점입니다."
(70초)

────────────────────────────────────────────────────────────
[Slide 09] ProtectSkills / MaliciousAgentSkillsBench (데이터셋)
────────────────────────────────────────────────────────────
"저희도 자체 데이터셋을 구축했습니다 — ProtectSkills /
MaliciousAgentSkillsBench. SkillsMP와 skills.rest에서 'malicious' 라벨이
붙은 157개 Skill을 모아서 14개 패턴으로 정적 분석했습니다.

상위 4개 패턴이 핵심 위협군입니다 — RCE 75%, Credential theft 64%,
Behavior Manipulation 64%, External Transmission 50%.
이 4개는 'credential 수집 → 외부 전송' 흐름을 만들어내는, 데이터셋의
대표 악성 chain입니다."
(75초)

────────────────────────────────────────────────────────────
[Slide 10] RCE 118개 — 단독 위협이 아닌 결합 수단
────────────────────────────────────────────────────────────
"RCE 118개를 더 들여다보면 흥미로운 패턴이 나옵니다.
대부분의 RCE가 단독으로 쓰이지 않고 다른 패턴과 결합됩니다 —
가장 많은 게 Credential theft + exfiltration RCE 48개고, 그다음이
Credential theft만 결합된 게 33개.

오른쪽 4-step 콤보가 가장 위험한 형태입니다 —
Behavior Manipulation으로 에이전트가 묻지 않고 실행하게 하고,
RCE로 payload를 실행하고, credential을 수집하고, 외부로 송출.

157개 중 약 30%가 이 4-step 콤보를 충족합니다 — 정상으로 보이는
Skill이 곧 완성형 공격이 되는 구조예요."
(75초)

[Slide 11] Privacy 영향의 두 축
────────────────────────────────────────────────────────────
"이게 결국 Privacy 문제로 귀결되는 이유입니다.

외부 클라우드 AI 사용까지 허용되는 추세입니다.
규제 완화 의도는 좋지만, AI로 인한 새로운 Privacy 위협 가이드라인은
따라잡지 못하고 있다는 게 문제예요."
(70초 — 8분 마크 근처)

[Slide 13] 최근 Agent 보안 연구 — 5개 흐름
────────────────────────────────────────────────────────────
"기존 연구를 5개 흐름으로 정리했습니다 —
01 Agent benchmark (AgentDojo, InjecAgent),
02 Information-flow defense (RTBAS, CaMeL),
03 Privilege control (Progent, Prompt Flow Integrity),
04 Instruction-data separation (ASIDE, Spotlighting, StruQ),
05 Skill supply-chain security (Snyk).

이 중 5번이 본 연구 영역과 가장 가깝지만, 기존 흐름은 모두 '공격

차단했나'를 비교한 연구는 거의 없습니다."

────────────────────────────────────────────────────────────

GAP 02 — 설치·적용 단계의 판단 분석.
GAP 03 — 설명과 실제 행동의 불일치.
나머지는 화면 참고하시고요. 모두 본 연구 설계에 직접 반영됩니다."

────────────────────────────────────────────────────────────
[Slide 15] 연구 질문 — 4가지 핵심 RQ
────────────────────────────────────────────────────────────
"위 공백을 정리한 4가지 RQ입니다.
RQ1 — 각 AI Agent가 악성 Skill을 얼마나 수용·차단하는가.
RQ2 — 차단 근거가 무엇이고, 그 설명이 실제 행동과 일치하는가.
RQ3 — Agent별 판단 차이는 어떻게 나타나는가.
RQ4 — 외부 scanner / proxy hook을 추가하면 ASR이 얼마나 감소하는가.

각 RQ는 측정 가능한 정량 지표와 1:1 매핑되어 있습니다."
(70초)
